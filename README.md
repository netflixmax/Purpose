# Purpose · 立志的递归算法

> 志不立，天下无可成之事。—— 阳明
> 为学日益，为道日损。—— 老子

一个把"立志"建模为**化繁为简的递归对话**的 CLI 产婆。
你带着一桩困惑进来，按选定的化简透镜逐层归本，直到当前陈述
能被你的直觉一眼判断——那就是你的志。

## 思路

- **递归 = 化繁为简**：每一步把当前困惑通过一把"透镜"化简一层，
  不是函数的复合调用，而是逐步归本的对话轨迹。
- **五把化简透镜**：归本（道德经）、反观（反者道之动）、定型（找原型族）、
  优势（演绎归 AI、直觉留己）、锚据（终止：能凭直觉判断了吗？）
- **产婆，非答主**：系统不替你拟志向。它追问、归类、提议化简；
  你用归纳与直觉接生那个一直在的志。
- **两套引擎，同一接口**：
  - `structural`：纯离线，只发出苏格拉底式追问；
  - `llm`：调 Claude（默认 `claude-opus-4-7`），给出候选化简与产婆评注。
- **副产品 = 干净数据**：每次会话写一行 JSONL 到
  `~/.purpose/journal.jsonl`，记录 seed → (lens, reduction)* → 志。

## 安装

```sh
pip install -e .
```

## 使用

```sh
# 离线、纯结构化
purpose

# LLM 辅助
export ANTHROPIC_API_KEY=sk-ant-...
purpose --engine llm
```

可选环境变量：
- `PURPOSE_MODEL`：覆盖默认模型（默认 `claude-opus-4-7`）。

## 开发

```sh
pip install -e ".[dev]"
pytest
```
